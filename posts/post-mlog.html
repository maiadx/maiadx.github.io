Writing a Rust Logger in a Weekend

View Project on GitHub

Features

  • Supports varying log levels
  • Colorized output
  • Optional file logging
  • Async mode
  • Multithreaded mode
  • Implements std::panic_hook to save crash information according to your configuration

Log Level Colors

Usage


info!("This is an info message.");
warn!("This is a warning.");
error!("This is an error: {}", 2);
crit!("Critical error, uh oh.");
success!("Operation successful! :)");

Initialization

You can initialize the logger with any log level; lower-priority logs will be automatically suppressed.
Priorities: Info (lowest) → Success → Warn → Error → Crit (highest priority) 


mlog::init(log_config);
// ...
mlog::shutdown(); // Safely shutdown the logger before exiting.

Configuration


let log_config = LogConfig {
    log_level: LogLevel::Info,                     // Defaults to logging everything
    application_name: "example application".to_string(),
    log_filepath: Some("logs/example"),            // Filepath for logs (default: None)
    console_flag: true,                            // Toggle console logging
    async_flag: true,                              // Async logger (defaults to false)
    multi_threaded_flag: true,                     // Single-threaded by default
    time_format: "%Y-%m-%d %H:%M:%S%.3f".to_string(), // Customizable time format
};

Additionally, if you build with the `performance` feature flag, all logs that are not errors or critical errors are removed at compile time (for release builds only).
